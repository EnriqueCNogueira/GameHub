O objetivo deste projeto é desenvolver o GameHub, um ecossistema completo de distribuição digital de jogos, inspirado na Steam. O sistema deve operar como uma plataforma robusta que integra três pilares principais: uma loja virtual (e-commerce), um gerenciador de biblioteca pessoal e uma rede social para gamers. O backend será uma API RESTful responsável por toda a lógica de negócios e persistência de dados, enquanto o frontend será uma aplicação web interativa que consome essa API para permitir aos usuários navegar, comprar, avaliar, etc.

Stack Tecnológica
Linguagem Principal: TypeScript.
Backend: Node.js com Express.js.
Banco de Dados: SQLite manipulado via Prisma ORM.
Frontend: HTML5, CSS3 e JavaScript puro.

Referência Arquitetural (Gabarito)

Modelagem do Banco de Dados
Usuario: "Informações de login (email, senha), dados de perfil (nome) e o saldo financeiro da carteira virtual."
Jogo: "O catálogo de produtos contendo título, descrição, preço, data de lançamento e referências às empresas responsáveis."
Desenvolvedor: Cadastro das empresas responsáveis pela criação técnica dos jogos.
Publicadora: Cadastro das empresas responsáveis pela distribuição e venda dos jogos.
Genero: "Categorias principais para classificação dos jogos (ex: RPG, Ação, Estratégia)."
Tag: "Etiquetas descritivas granulares para filtrar jogos (ex: ""Zumbis"", ""Mundo Aberto"")."
Transacao: "O registro histórico e imutável de uma compra finalizada, contendo a data e o valor total pago."
ItemTransacao: Tabela de junção que detalha exatamente quais jogos estavam inclusos em uma transação específica.
Biblioteca: "A relação de ""posse"" que vincula um usuário aos jogos que ele já comprou, incluindo o tempo de jogo registrado."
Carrinho: Armazenamento temporário dos jogos selecionados pelo usuário para uma compra imediata (pré-checkout).
Wishlist (Lista de Desejo): Jogos que o usuário salvou para monitorar ou comprar no futuro.
Analise: Reviews feitos pelos usuários contendo uma nota (avaliação numérica) e um comentário de texto.
Amizades: "Relacionamento social auto-referenciado entre usuários, guardando o status do convite e a data."
JogoGenero / JogoTag: Tabelas associativas que permitem que um único jogo possua múltiplos gêneros e múltiplas tags simultaneamente.

Funcionalidades Detalhadas do Sistema
O projeto final deve contemplar as seguintes funcionalidades:
Catálogo e Navegação: Listagem de jogos com filtros por Gênero e Tag, busca por nome, desenvolvedora, publicadora.
Ciclo de Compra Completo:
    Adicionar/Remover jogos do Carrinho.
    Adicionar jogos à Lista de Desejos.
    Checkout de Transação: Converter itens do carrinho em uma Transação fechada e mover os jogos para a Biblioteca do usuário.
Gestão de Biblioteca: Visualização exclusiva dos jogos adquiridos pelo usuário logado.
Sistema Social: Enviar solicitações de amizade e listar amigos adicionados.
Avaliações: Usuários que possuem um jogo podem criar uma Análise (Review) que aparecerá na página do produto.
Roteiro de Execução

Desenvolva o projeto seguindo esta ordem lógica:
schema.prisma: Defina todas as tabelas e relacionamentos citados acima.
domain.ts: Crie as interfaces TypeScript equivalentes para garantir tipagem em toda a aplicação.
repository.ts: Implemente as funções de CRUD e regras de negócio (ex: addFriend, checkoutCart, getLibrary, postReview) usando o PrismaClient.
server.ts: Configure o Express e crie rotas para expor cada função do repositório (ex: POST /cart/checkout, GET /games/search).
Frontend (index.html, style.css, main.js): Construa a interface visual baseada no protótipo (Loja escura, cards de jogos) e integre com a API usando fetch.
